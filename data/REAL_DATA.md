# Real data in CellPilot

CellPilot is evaluated on a **real, third-party fed-batch dataset** — not data it
generated itself.

## Wired-in dataset: `data/ieks_9batches/`

Nine independent fed-batch mammalian-cell runs (15 timepoints each, 0–14 days) with
VCD, glucose, lactate, glutamine, ammonium, product, and the **actual feed-bolus
schedules**.

- **Source:** `dkisdongkyu/ieks-specific-rate` (GitHub), `data/simulated_9batches`,
  from the manuscript *High-Fidelity Cell-Specific Kinetic Rate Estimation in Fed-Batch
  Mammalian Cell Cultures Using an Iterated Extended Kalman Smoother* (submitted to
  *Biotechnology and Bioengineering*).
- **Loader:** `cellpilot/realdata.py` (`load_ieks_batches`) — handles unit reconciliation
  to CellPilot's schema (days→hours, L→mL, mg/L→g/L; VCD 10⁹/L ≡ 10⁶/mL).
- **Used by:** `scripts/evaluate.py` (hybrid vs mechanistic vs pure-ML, leave-one-out CV)
  and `scripts/active_learning.py` (oracle calibration).

**Why this matters for credibility:** the data was generated by a *different group* with
a *different model*. Fitting CellPilot's Monod model to it and learning a residual is a
genuine cross-model test — the mechanism mismatch the hybrid corrects is real, not
something CellPilot injected. (These runs are themselves model-simulated, not wet-lab
measurements; truly clean, downloadable, physical-unit wet-lab CHO time series are rarely
public — which is itself the data-fragmentation problem Tolemy's Orbit attacks.)

## Bringing your own data

The ingestion layer (`cellpilot/ingest.py`) absorbs messy real exports:

```python
from cellpilot.ingest import ingest_csv
run = ingest_csv("path/to/run.csv", use_llm=True)  # use_llm needs ANTHROPIC_API_KEY
```

The heuristic mapper knows synonyms for VCD, viability, glucose, glutamine, lactate,
ammonia (NH4+), titer (mAb), and volume; the optional LLM mapper resolves the rest.
One row per timepoint, one column per variable, plus a time column (`Culture Time (h)`,
`Day`, `Elapsed (h)`…; day→hour handled). Multiple runs → one file per run or a `run_id`
column.
